logback-spring.xml の中では :- でなく : を使用しましょう。

on 2020-01-28

#spring-boot  #logback 

要旨

logback-spring.xml の中では :- でなく : を使用しましょう。

解説

Spring Bootのリファレンスに次のような記述があります:

If you want to use a placeholder in a logging property, you should use Spring Boot’s syntax and not the syntax of the underlying framework. Notably, if you use Logback, you should use : as the delimiter between a property name and its default value and not use :-.

— 4.6. Custom Log Configuration

にもかかわらず、結構 logback-spring.xml の中で :- を使用しているサンプルが検索すると見つかります。

この点が少し気になったので調べてみました。

Gitリポジトリで履歴をたどったところ、次のissueで追加された文章でした:

  • Is DefaultLogbackConfiguration correct with ${LOG_LEVEL_PATTERN:-%5p} #4731

logback-spring.xml の中のプレースホルダは(logback.xml とは異なり) Logback でなく Spring が処理するので、 Logback記法ではなく Spring記法を使うべし、ということのようです。

:- はLogbackのデフォルト値指定、 : はSpring記法でのデフォルト値指定ですね。
